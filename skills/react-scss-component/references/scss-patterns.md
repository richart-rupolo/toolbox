# SCSS Patterns

## Prefer

- semantic tokens
- local root classes
- clear state classes
- feature-specific styles staying near the component

## Avoid

- hardcoded hex colors in TSX
- utility-class sprawl without a system
- visual class names
- recreating button or input primitives in each feature

## Example Naming

- `language-selector`
- `tool-actions`
- `status-message`
- `page-header`
